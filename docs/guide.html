field guide · built 2026-07-10

pi-orchestra

Your expensive AI assistants (Claude Code, Codex) stay the brains. A cheap giant-context worker (MiniMax M3 via the pi CLI) does the heavy lifting. Every delegated job is recorded, quota-guarded, and killable — and you watch it all in one terminal dashboard: orc top.

01 · the idea

Why this exists

Claude and Codex run on your paid subscriptions — great judgment, expensive tokens. MiniMax M3 has a 1,000,000-token context window and costs about $0.30 / $1.20 per million tokens (a full question–answer round trip we measured cost $0.00014). So the rule is simple:

brains decide · workers grind · you watch & veto

Reading 50 files, summarizing a huge log, a mechanical refactor across a repo — that's worker food. Judgment calls, small edits, anything needing back-and-forth with you — the brain keeps those.

03 · in practice

What happens when you say things

"fix this typo in app.py"

Nothing special. Small task → the brain does it itself. No workers, no registry entries, zero orchestration overhead.

"summarize all 80 files in src/"

Auto-delegation. The pi-delegate skill kicks in: quota is checked, one MiniMax worker grinds through it, the brain verifies the result before using it.

"orchestrate: audit this whole repo"

Full orchestration — only on this keyword. Quota report → task split into ≤3 parallel background workers → brain polls, verifies, synthesizes → per-worker report.

The word orchestrate (or orchestrated) is the only trigger for multi-worker mode. Ordinary prompts — even heavy ones — never spin up the swarm.

04 · the fuel gauge

Quota guard — the brain knows how much MiniMax you have left

Your MiniMax coding plan is metered by two rolling windows. Before every delegation, orc asks MiniMax's own API (/v1/token_plan/remains) how much is left — real numbers, not guesses:

5-hour window
99%
weekly window
49%

(live values at the time this guide was generated)

Three thresholds, three behaviors

RemainingWhat orc doesWhat you see
> 25% Runs normally. Nothing — it just works.
≤ 25% Runs, but warns. Brains are required to repeat the warning to you verbatim. ORC WARNING: MiniMax quota low…
≤ 10% Refuses to start the worker (exit code 3). Only --force overrides, and brains may not use it without your say-so. ORC BLOCKED: … Use --force to override.

Thresholds live in ~/.orchestra/config.json (warn_pct, block_pct). If the quota API is unreachable, work is never blocked — you just get an honest quota unknown note.

06 · your question, answered

Is it "on" for Claude and Codex whenever I run them?

Yes — from every new session, automatically. Nothing to launch, no daemon, no per-project setup. Here's exactly what loads where:

Automatic (no action needed)

  • Claude Code reads ~/.claude/skills/pi-delegate and …/orchestrate at session start — they're global, so every project gets them. (Symlinks into the repo, so git pull updates behavior.)
  • Codex reads ~/.codex/AGENTS.md every session — the pi-orchestra block travels with it.
  • orc sits at ~/.local/bin/orc, already on PATH for both brains' shells and yours.
  • Quota gate + registry + watchdog are inside orc itself — any caller gets them for free.

Only when you want it

  • deleg8 / pi-rpc are zsh functions for your interactive terminal (new terminals from now on; existing ones need source ~/.zshrc).
  • Brains' tool shells are non-interactive, so they don't see those functions — their skills tell them to call orc run … --brain claude directly. Verified working.
  • orc top — run it whenever you want the dashboard; it's a viewer, nothing depends on it running.
  • The orchestrate swarm — fires only when you type the keyword.
One caveat: sessions that were already open before installation (like the one that built this) don't have the skills loaded — skills are read at session start. Anything started from now on has the full stack.

08 · under the hood

One run, on disk

No database, no server. Each delegation is a folder you can read with cat — which is also exactly what the dashboard does:

~/.orchestra/runs/20260710-023226-reply-with-the-single-b7d2/
├── meta.json      # the record ↓
├── output.log     # everything the worker printed
└── inbox/         # control messages (a kill request is just a JSON file)

meta.json:
{
  "id": "20260710-023226-reply-with-the-single-b7d2",
  "task": "Reply with the single word: PONG",
  "brain": "human",          // or "claude" / "codex" — who ordered it
  "status": "done",          // starting → running → done | failed | killed | orphaned
  "pid": 41233,
  "started_at": "2026-07-09T21:02:26+00:00",
  "ended_at":   "2026-07-09T21:02:41+00:00",
  "exit_code": 0,
  "tokens": { "input": 84, "output": 19, "total": 1639, "cost_usd": 0.00014 }
}
StatusMeaning
runningWorker is alive right now (PID is checked, not trusted).
doneFinished cleanly; output is in output.log.
failedWorker exited with an error — including exit 124, the idle watchdog killing a stalled worker.
killedYou (or a brain) killed it via orc kill, the TUI, or Ctrl+C.
orphanedMachine slept or the process vanished; reconciled automatically on the next orc list.

09 · guarantees

What it will never touch

The whole install is additive, and it was checksum-verified before and after:

  • ~/.pi/agent/auth.json & settings.json — untouched. Plain pi and your pix launcher behave exactly as before.
  • ~/.claude/settings.json — untouched. Plain claude stays on your Anthropic subscription.
  • ~/.codex/config.toml — untouched. Plain codex stays on OpenAI.
  • The only writes: two marked blocks (in ~/.zshrc and ~/.codex/AGENTS.md, both backed up as *.pi-orchestra.bak first) plus symlinks. ./uninstall.sh reverses all of it.
  • Your API key stays in the macOS Keychain — read at runtime, never copied anywhere.

10 · battle scars

Built by using itself — what the dogfooding taught us

MiniMax M3 drafted most of orc's own code while Claude reviewed and integrated — the exact advisor pattern the system implements. Every lesson below is now baked into the code or the skills:

--offline, always

pi's startup network check once hung a worker for 5 minutes. Every orc-spawned worker now passes it.

stalls are normal

~50% of long-prompt calls stalled or aborted that day. Hence the idle watchdog: no output for idle_timeout_sec (default 300s) → killed, exit 124.

--thinking low for drafts

Default thinking caused multi-minute stalls; low took 10–47s with equal quality on well-specified tasks.

review is not optional

Worker drafts were good — and still shipped a real bug almost every time (an invented JSON key, a broken docstring). Brains must verify before acting.

rpc dies with stdin

pi's streaming mode exits the moment stdin closes — naïve one-shot pipes silently produce nothing. orc holds stdin open until agent_end.

exact costs, free

pi's final event carries real token usage and dollar cost; orc records it in meta.json instead of estimating.
