Visual Identity System / rev 2026.07
THEME

pi⁠-⁠orchestra

▸ conductor · bench · baton

A design language for the terminal that orchestrates AI coding agents. One expensive brain plans and delegates to a bench of cheap workers. Sessions are durable — panes survive detach and crash. Everything below renders on a character grid via ratatui: box-drawing borders, block-element bars, braille motion, and monospace type where weight is simulated with color, bold, and reverse-video.

00

The metaphor

orchestra · conductor · bench · baton
◆ conductor

The one expensive brain. Plans, decomposes, and delegates. Rendered in brain-accent, always the top-left anchor, framed with a heavier active border.

● the bench

The pool of cheap workers waiting for dispatch. Each seat resolves a harness on PATH. Rendered in worker-accent, filled circle when seated, hollow when idle.

╺━━╸ the baton

The filament that connects conductor to worker. Idle = dim dotted rule; a pulse travels along it whenever a pane produces output. The signature motion of the app.

⏻ durable session

Panes survive terminal detach and crash. A session is a shelf of reattachable cards, each carrying its own health, worker count, and last-seen time.

01

Three themes

flagship + two anchors

Each theme is a full Base16-style semantic slot map — color encodes meaning, never decoration. Remove all color and the interface stays usable through layout, glyphs, and reverse-video. True color enhances; it never creates the hierarchy.

nocturne FLAGSHIP

Stage at night. Near-black blue house lights, cool teal for the conductor, periwinkle for the bench, warm gold spotlight for confirmations. Theatrical, premium, dense.

02

Color token sheet

hex + nearest ANSI-256 fallback

Every semantic slot, all three themes. The ANSI-256 column is the graceful fallback when truecolor is not detected. Widget code references the slot name, never a hex literal — themes are swapped by remapping the slots.

SEMANTIC SLOT USAGE nocturne ember phosphor

ANSI-256 codes are the nearest xterm cube/greyscale index; verify against your emulator's palette. Respect NO_COLOR — when set, fall through to the 16-color / phosphor tier below.

03

Type & iconography

monospace · nerd-font glyphs · ASCII fallback
PRIMARY FACE
JetBrains Mono

Chosen for a true 1:1 cell, unambiguous 0/O · 1/l/I, crisp box-drawing joins, and a full Nerd Font patch. Ships free (OFL). Recommend the Mono nerd variant so every icon forces single-cell width.

weight is simulated, never a font-weight axis:
regular body · bold emphasis ·  reverse  selection · dim metadata
Alternatives: Berkeley Mono (premium, paid) · Iosevka (tall/condensed, dense dashboards) · IBM Plex Mono (technical). Phosphor pairs beautifully with a VT-style face if you have one.
TYPE SCALE — CELL ROWS, NOT PX
Aa Screen titlebold + brain · 1 row
Aa Panel headerbold + fg · in border
Aa Body / content — 80%regular fg.default
Aa metadata · timestampsdim fg.muted
Hierarchy recipe: 80% of content in fg.default, headers in bold+emphasis, metadata dim, status in its semantic color, accent reserved for interactive/active.
GLYPH REGISTER — every state has a symbol, so color is never load-bearing alone

Screen · STAGE

      live panes · the baton · zoom / swap
    

    
The main live view: one conductor pane and up to three worker panes, each a bordered PTY card. The active card carries a heavier, accented frame — the same visual language as zellij's focused pane. Baton filaments run from the conductor to each worker and pulse whenever a pane produces output.

      TASK CONFIRMED · CONDUCTOR DOWN
    

    

      

        
✓ TASK CONFIRMED
A dispatch lands. Not a text swap — a tactile stamp: the row flashes to reverse-video for 1 frame, the glyph stamps in scaling down, then settles to a steady confirmed badge with a gold underline.

⏻ CONDUCTOR DOWN
The brain process died. This must read as calm and recoverable, not alarming — muted coral, never red-alert; a slow breath, not a blink; elapsed time and a clear R to recover. Workers hold their last state.

        

          

        

      

    

  


  
  

    

      09
      
Degradation tiers

      design in layers — each stands alone
    

    
Start monochrome (usable?), add 16 ANSI colors (readable?), layer truecolor (beautiful?). And two orthogonal switches: reduced motion, and a minimum viewport. Below 80×24, everything collapses to a resize prompt.

    

  


  
  

    

      10
      
Status bar & keys

      context-sensitive · vim-fluent
    

    
One persistent line at the bottom shows the mode on the left and immediate context actions on the right — the same discipline as zellij and gitui. It changes per screen; it shows what's actionable now, never the whole keymap.

Borrowed patterns

        
    
          
  • gitui — semantic RON theme slots (selection_fg, command_fg); theme-aware, light/dark-adaptive; context help footer.

    • 
          
  • atuin — full-screen fuzzy search, columns [time · duration · exit · cmd], exit-code badges, filter cycling, reduced-motion setting.
    • 
          
  • television — channels, real-time preview pane, nucleo fuzzy match highlighting, minimal chrome.
    • 
          
  • Nerd Fonts — PUA icon glyphs + native Braille block; use Mono variants for single-cell width.
    • 
        

      

      

        
Principles applied

        
    
          
  • Semantic color — remove all color, stay usable through layout, glyphs, reverse-video.
    • 
          
  • True color enhances, never creates, the hierarchy — the 16-color / phosphor tier must stand alone.
    • 
          
  • Never color alone — every state pairs with a glyph and, where it matters, a text label.
    • 
          
  • Context-sensitive footer — show what's actionable now, not the entire keymap.
    • 
          
  • Async everything, cancel with Esc; spinners for indeterminate work.
    • 
          
  • Min viewport 80×24; below it, a resize prompt.
