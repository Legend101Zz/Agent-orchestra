# orc top v2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the stock-looking `orc top` with a btop-style instrument-cluster dashboard plus a Posting-style session drill-in screen, backed by an honest data layer (exact tokens/cost, session grouping, brain-side usage metrics).

**Architecture:** Data layer first — `runner._exec` moves to `pi -p --mode json` for exact usage; `meta["session"]` groups runs; a new `orc_pkg/metrics.py` aggregates worker registry + brain JSONL logs. Then `orc_pkg/tui.py` becomes the `orc_pkg/tui/` package: pure, unit-testable render functions (gradient meters, braille sparklines, flow DAG, conversation parsing, session grouping) composed by a themed Textual app with a Dashboard screen and a Session detail screen.

**Tech Stack:** Python 3.14, Textual 8.2.8 (already pinned in `.venv`), stdlib only otherwise. No new dependencies (netext rejected; DAG hand-rolled).

## Global Constraints

- Never modify `~/.pi/agent/*`, `~/.claude/settings.json`, `~/.codex/config.toml`.
- Registry schema changes must be backward compatible — old runs without `session`/exact-token fields must still render everywhere.
- TUI stays read-only against the registry; kills/new tasks shell out to `python -m orc_pkg kill/run`.
- Every pi invocation keeps `--offline`.
- All 26 existing tests must keep passing: `.venv/bin/python -m pytest -q`.
- Zero `#0178d4` (stock Textual blue) anywhere; every color comes from a theme token in `orc_pkg/tui/theme.py`.
- Key conflict resolution: vim `j`/`k` navigate; kill is `x` (spec lists both `k` meanings; nav wins).
- Commits: small, conventional prefixes (`feat:`, `refactor:`, `test:`, `docs:`). Push to `origin main` at the end; no force-push.
- Run commands with the repo venv: `.venv/bin/python`.

**Interfaces:**
- Produces:
  - `metrics.WORKER_PRICE = (0.30, 1.20)` and `metrics.BRAIN_PRICE = {"claude": (3.0, 15.0), "codex": (1.25, 10.0), "human": (3.0, 15.0)}` — USD per 1M tokens (in, out); brain prices are Sonnet / GPT-5-class list prices, labeled "API-equivalent".
  - `metrics.worker_stats(runs: list[dict]) -> dict` → `{"runs": N, "by_status": {...}, "exact": {"runs", "input", "output", "cache_read", "total", "cost_usd"}, "estimated": {"runs", "total"}, "by_brain": {brain: {"runs", "total", "cost_usd"}}, "by_session": {sess: {"runs", "total", "cost_usd", "status"}}, "by_day": {"YYYY-MM-DD": {"runs", "total"}}}`. A run is *exact* iff `tokens` has `total`; otherwise its `estimated_total` counts as estimated. `cost_usd` for exact runs = recorded `cost_usd` if present else `input/1e6*0.30 + output/1e6*1.20`. Day key from `started_at[:10]`. Session status = worst of members (running > failed > killed > orphaned > done).
  - `metrics.delegated_value(runs) -> dict` → `{"worker_cost_usd", "brain_equiv_usd", "saved_usd", "multiple", "exact_share"}`; brain-equiv prices each run's tokens at `BRAIN_PRICE[run brain]`; estimated-only runs price `estimated_total` as all-input; `exact_share` = exact-token fraction (honesty signal).
  - `control.cmd_stats(args)` — `orc stats [--json]`.
